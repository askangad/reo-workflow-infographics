Convert your OSS users into enterprise customers
Spin up a personalised landing page per target account — fed by their own developer-usage data — and use it as the wedge for the enterprise conversation.
1
Build the target list in Reo
Create a segment of accounts you want a 1-pager for — your in-market OSS users, your hot signals cohort, your top-of-funnel ICP. This list is the universe everything else operates on.
Account universe
Reo.Dev — segments & account lists
2
Pull raw usage data via the developers API
For every company in the segment, hit the Reo developers API to pull the raw activity feed — page visits, doc reads, sandbox runs, CLI events — and the identified-developer roster.
Data source
developers.reo.dev API
Reo.Dev — activity feed + dev roster
3
LLM parses raw data into landing-page sections
Hand the raw API response to an LLM and ask it to bucket the data into the sections your landing-page template needs — top use cases, page-visit breakdown, active-dev count, estimated engineering hours, build-vs-buy framing, per-dev highlights. One pass, structured JSON out.

4
Store the structured per-account data
Write the LLM's structured output to whatever backend your landing-page builder reads from — one row per account, with all the page-section fields as columns. This is what your template binds against at publish time.
Backend
Postgres / Supabase
Airtable
Google Sheet / CSV
5
Publish one 1-pager per account
Bind the per-account row to your landing-page template and deploy. Each account gets a unique shareable URL you can drop into LinkedIn DMs and emails — and every page is a personalised, data-backed conversation starter.
